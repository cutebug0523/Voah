# 粗暴混剪测试记录

## 2026-06-02：花西子气垫 rough v1

目标：

```text
用同一产品的 15-20 条素材做一次粗暴混剪闭环：
素材理解/向量检索 -> 自动脚本 -> TTS 配音 -> 字幕 -> FFmpeg 成片
```

本轮先固定目标：

```text
产品：花西子气垫
成片：45 秒左右，9:16，带货种草风格
结构：开场钩子 -> 产品亮相 -> 遮瑕/质地 -> 上脸效果 -> 套装福利 -> CTA
素材池：/Users/noah/混剪/原片 中最短的 18 条竖屏素材
```

这版的向量/检索维度必须包含：

- `product_name`：产品名，第一版固定为“花西子气垫/花西子好气色气垫/四效合一气垫”。
- `cta_intent`：直播间、福利、优惠券、现在冲、点击进入等 CTA 信号。
- `visual_role`：产品亮相、开箱、质地、遮瑕、上脸、礼盒、生活方式。
- `transcript_or_ocr`：来自 Omni/字幕/口播的文本。
- `remix_role`：开场钩子、卖点承载、功能证明、效果展示、价值强化、CTA 收尾。

注意：

- 这是粗测，不追求最终剪辑审美。
- 先看“跨素材重组能不能成立”，再决定后续精修索引结构。

核验结果：

- 成片规格：720x1280，30fps，约 72.15 秒，约 25MB。
- 素材覆盖：时间线使用 18 个片段，来自 18 条不同源素材。
- 产品维度：142 个有效 chunk 的 `enriched_text` 都写入“花西子气垫”及别名。
- CTA 维度：29 个 chunk 带 CTA 信号，集中在 `CTA收尾` 角色。
- 角色分布：开场钩子 18、产品亮相 26、质地/遮瑕证明 22、上脸效果 28、套装/价值感 19、CTA收尾 29。
- 视频向量化：使用 `qwen3-vl-embedding`，本轮 18 条素材产出 142 个有效 chunk；3 个过小 chunk 因 DashScope 视频体积下限失败。
- 配音：使用 macOS `say -v Tingting` 做粗测 TTS。
- 字幕：当前 ffmpeg 没有 `subtitles`、`drawtext`、`ass` filter，实际采用 Pillow 生成透明 PNG 字幕，再逐片 overlay 后 concat。

本轮粗糙点：

- 成片可验证“同产品、跨素材重组”闭环成立，但不是成品级剪辑。
- 原素材大多自带硬字幕，新字幕叠加后会和原字幕撞位，这是后续字幕策略必须单独解决的问题。
- TTS 比较机械，而且为了贴合配音长度把画面扩展到约 72 秒，已经偏离原先 45 秒目标。
- 角色和卖点目前主要靠启发式 enrichment，不是 Omni 逐片精确重打标签。
- 时间线字段里实际角色字段是 `role` / `source_role`，不是 `remix_role`；后续 skill 化时需要统一命名。
