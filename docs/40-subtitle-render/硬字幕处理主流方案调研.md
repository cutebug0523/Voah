# 硬字幕处理主流方案调研

调研日期：2026-05-30

## 1. 结论摘要

主流方案不是“替换硬字幕”，而是根据素材情况选择不同处理等级：

```text
软字幕轨
  -> 直接移除/替换字幕轨

硬字幕但不影响画面
  -> 保留原字幕，新字幕避让

硬字幕在底部边缘
  -> 裁剪、遮挡、模糊、重构画布

硬字幕必须清除
  -> OCR/检测字幕区域 + 视频 inpainting

高质量人工片
  -> After Effects / Runway / 专业 VFX 工具手工 mask + 内容感知填充
```

对我们的混剪工作台来说，第一版最稳的是：

```text
检测硬字幕
  -> 作为素材元数据入库
  -> 渲染时支持 keep / crop / cover / blur
  -> 暂不默认自动 inpainting
```

后续如果用户明确要求“去字幕干净版”，再接入 VSR/ProPainter/LaMa 这类本地或云端修复链路。

## 2. 方案 A：软字幕轨移除或替换

适用：

- MKV/MP4 内有独立字幕流。
- 或有外部 `.srt` / `.ass` 字幕文件。

处理方式：

- 用 `ffprobe` 检查 subtitle stream。
- 用 FFmpeg `-sn` 或 `-map` 排除字幕流。
- 重新挂载新的 `.srt` / `.ass`。

优点：

- 快。
- 无画质损失。
- 不需要 AI。

缺点：

- 对硬字幕无效。

资料：

- FFmpeg 官方文档说明了 stream selection、subtitle stream 和 `-map` 控制方式。
- URL：https://ffmpeg.org/ffmpeg.html

## 3. 方案 B：裁剪 / 遮挡 / 模糊 / 避让

适用：

- 原字幕固定在底部。
- 不追求完全恢复原画面。
- 目标是混剪发布，而不是修复母版。

处理方式：

- `crop`：裁掉底部字幕区。
- `drawbox`：用黑条、渐变条、信息栏遮住原字幕。
- `boxblur`：对字幕区域局部模糊。
- `overlay`：用新字幕条覆盖旧字幕。
- 新字幕避让：新字幕放到中下、上方或侧边。

优点：

- 最稳定。
- 成本最低。
- FFmpeg 可实现。
- 很适合短视频混剪。

缺点：

- 不是真正去字幕。
- 画面可能损失或变“加工感”。

资料：

- FFmpeg filter 文档包含 crop、drawbox、boxblur、overlay 等基础滤镜。
- URL：https://ffmpeg.org/ffmpeg-filters.html

## 4. 方案 C：OCR/文本检测 + 视频 Inpainting

这是当前“自动去硬字幕”的主流技术路线。

基本流程：

```text
抽帧
  ↓
检测字幕区域 / OCR
  ↓
生成 mask
  ↓
视频 inpainting
  ↓
时序一致性处理
  ↓
重新编码
```

常见模型/算法：

- STTN：时空 Transformer 视频修复。
- LaMa：单帧图像修复，适合静态背景/残留修复。
- ProPainter：更强的视频 inpainting，适合运动和复杂背景，但更慢。
- OpenCV：简单快速，但质量较低。

主流开源工具：

- `YaoFANGUK/video-subtitle-remover`
- `SysAdminDoc/VideoSubtitleRemover`

这些工具通常支持：

- 自动检测字幕区域。
- 手动指定字幕区域。
- 多种修复模式，如 `sttn-auto`、`sttn-det`、`lama`、`propainter`、`opencv`。
- GPU / Docker / GUI / CLI。

优点：

- 能真正尝试清除硬字幕。
- 可以批处理。
- 适合素材清洗。

缺点：

- 慢。
- 需要 GPU 才舒服。
- 复杂背景、字幕压脸、字幕大面积遮挡时容易失败。
- 容易闪烁、糊、涂抹。
- 每次都要重编码。

资料：

- VSR GitHub：https://github.com/YaoFANGUK/video-subtitle-remover
- VSR Pro GitHub：https://github.com/SysAdminDoc/VideoSubtitleRemover
- STTN 论文：https://arxiv.org/abs/2007.10247
- ProPainter 论文：https://openaccess.thecvf.com/content/ICCV2023/papers/Zhou_ProPainter_Improving_Propagation_and_Transformer_for_Video_Inpainting_ICCV_2023_paper.pdf

## 5. 方案 D：专业剪辑软件 / 云端 AI 对象移除

代表：

- Adobe After Effects Content-Aware Fill
- Runway Inpainting / Remove from Video
- 各类在线视频 caption remover / watermark remover

处理方式：

- 人工刷 mask。
- 工具跟踪 mask。
- 生成内容感知填充。
- 人工预览并补关键帧。

优点：

- 对少量关键片段可控。
- 适合人工精修。
- 比纯自动更容易达到发布质量。

缺点：

- 不适合全自动批量。
- 需要人工干预。
- 云端工具可能有时长、文件大小、隐私、价格限制。
- 对长视频和复杂运动仍不稳定。

资料：

- Adobe Content-Aware Fill：https://helpx.adobe.com/uk/after-effects/using/content-aware-fill.html
- Runway Inpainting：https://help.runwayml.com/hc/en-us/articles/19155664495379-Inpainting

## 6. 方案 E：最新研究方向：Mask-Free / 端到端去字幕

2026 年已有一些研究开始尝试不用显式 mask，直接做端到端字幕擦除。

代表：

- CLEAR：Context-Aware Learning with End-to-End Mask-Free Inference for Adaptive Video Subtitle Removal
- SEDiT：Mask-Free Video Subtitle Erasure via One-step Diffusion Transformer

优点：

- 理论上减少 OCR/mask 错误传播。
- 更接近“一步去字幕”。

缺点：

- 研究阶段为主。
- 未必有可直接生产接入的稳定开源实现。
- 算力和一致性仍需验证。

资料：

- CLEAR：https://arxiv.org/abs/2603.21901
- SEDiT：https://arxiv.org/abs/2605.14894

## 7. 对我们项目的建议

第一版不要把 hard subtitle removal 当主功能。

建议分两层：

### 7.1 成片渲染层

必须支持：

- `keep`：保留原硬字幕，新字幕避让。
- `crop`：裁掉原硬字幕区。
- `cover`：遮挡原硬字幕后重上新字幕。
- `blur`：局部模糊原字幕区。

### 7.2 素材清洗层

后续可选支持：

- `inpaint-vsr`：调用 VSR 类工具。
- `inpaint-cloud`：调用云端对象移除工具。
- `manual-vfx`：导出给 AE / Runway / DaVinci 人工处理。

## 8. 混剪场景的实际选择

素材本身有字幕时，混剪不一定要强行去掉。

更实用的策略：

- 情绪类/剧情类素材：保留原字幕，把它当剧情信息。
- 解说类成片：新字幕放中下方，原字幕保留在底部。
- 信息冲突明显：遮挡或裁剪原字幕。
- 重要封面/开头 3 秒：必要时人工或 AI inpainting 精修。
- 批量素材库：先记录硬字幕区域和 OCR 文本，不急着清洗。
